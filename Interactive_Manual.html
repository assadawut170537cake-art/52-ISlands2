🏢 ภาพรวมระบบ Smart Worksite

ระบบ Smart Worksite Dashboard เป็นแพลตฟอร์มสำหรับจัดการลงเวลาทำงานพนักงาน, วิเคราะห์ข้อมูลด้วย AI, และควบคุมการดำเนินงานของไซต์ก่อสร้าง ออกแบบมาเพื่อทำงานร่วมกับ Google Sheets เป็นฐานข้อมูลหลัก

📋 คุณสมบัติหลัก

  • ลงเวลาพนักงาน — บันทึกเข้า-ออกงาน, คำนวณ OT อัตโนมัติ, แยกตามไซต์งาน
  • LINE Bot Integration — ลงเวลาผ่าน LINE กลุ่ม ด้วยข้อความหรือรูปภาพ
  • AI ประมวลผล — Gemini AI ช่วยวิเคราะห์ข้อความ, แปลงเสียง, ตรวจ PPE
  • Dashboard สถิติ — กราฟสถิติ OT, ชั่วโมงสะสม, PPE Compliance
  • คลังข้อมูล — ค้นหา/ส่งออก CSV ประวัติการลงเวลาย้อนหลัง
  • ระบบ Admin — จัดการพนักงาน CRUD, ตั้งค่า Config แบบ Real-time

🔐 ระบบสิทธิ์ผู้ใช้

ระบบจะแสดงอีเมล Google ของผู้ใช้งานปัจจุบันที่ Header และ Sidebar โดยตรวจสอบสิทธิ์ Admin จากรายการอีเมลที่กำหนดใน Config.gs

💡
ผู้ใช้ที่ไม่ใช่ Admin จะเห็น Badge 👤 User และไม่สามารถเข้าถึงเมนู "จัดการพนักงาน" หรือ "AI Admin" ได้

📱 วิธีลงเวลาผ่าน Web App

ขั้นตอนการบันทึกเวลาเข้า-ออกงานผ่าน Dashboard Web App:

1

เปิดหน้า Dashboard → คลิก "บันทึกเวลา"

หรือคลิกแท็บ "ลงเวลา" ที่แถบเมนูด้านล่าง

2

กรอกข้อมูลไซต์งาน

เลือกวันที่, ไซต์งาน, เวลาเข้า-ออก ระบบจะคำนวณชั่วโมงปกติและ OT ให้อัตโนมัติ

3

เลือกแคมป์ที่พัก → ติ๊กเลือกพนักงาน

เลือกแคมป์ที่พักจาก Dropdown แล้วคลิกเลือกพนักงานที่ต้องการลงเวลา

4

กดปุ่ม "บันทึกข้อมูลลงชีตรายวัน"

ข้อมูลจะถูกส่งไปบันทึกลงไฟล์ Google Sheets ประจำเดือนโดยตรง พร้อมแสดงผลลัพธ์

⏰ การคำนวณชั่วโมงอัตโนมัติ

ช่วงเวลาประเภทตัวอย่าง
ก่อน 08:00OT เช้า06:00-08:00 = OT 2 ชม.
08:00-17:00ชั่วโมงปกติหักพักเที่ยง 1 ชม. = 8 ชม.
12:00-13:00OT เที่ยง (ถ้าไม่พัก)ระบุ "OT เที่ยง" = OT 1 ชม.
หลัง 17:00OT เย็น17:00-19:00 = OT 2 ชม.
⚠️
การลงเวลาย้อนหลังเกินโควต้าที่กำหนด (ค่าเริ่มต้น 2 วัน) จะถูกระบบปฏิเสธ ยกเว้น Admin

💬 การลงเวลาผ่าน LINE Bot

พนักงานสามารถลงเวลาผ่าน LINE กลุ่ม โดยพิมพ์ข้อความตามรูปแบบที่ระบบ Regex กำหนด:

📌 รูปแบบข้อความมาตรฐาน

#18/05/2026
พระราม3 เข้า สวนหลวง
08.00-17.00
ทั้งหมด 2 คน
1. ออง / ทำงาน
2. สมชาย / ทำงาน

📌 รูปแบบพร้อม OT เที่ยง

#18/05/2026
บางละมุง เข้า ปรานบุรี
08.00-19.00 OT เที่ยง
ทั้งหมด 3 คน
1. วินวิน / ทำงาน
2. โมนัน / ทำงาน / OT เที่ยง
3. แทน / เข้าฟอร์ม

⚠️ กฎเหล็กการส่งข้อความ

  • เริ่มต้นด้วย # — ทุกข้อความลงเวลาต้องเริ่มด้วยเครื่องหมาย # ตามด้วยวันที่
  • ชื่อพนักงานต้องตรง (Exact Match) — ระบบปิดการเดาชื่อ (Fuzzy) เพื่อป้องกันข้อมูลลงผิดคน
  • รูปแบบวันที่ — รองรับ DD/MM/YYYY, DD.MM.YY, DD-MM-YY
  • เวลาใช้จุด — เช่น 08.00-17.00 (ไม่ใช่โคลอน)
  • ที่พัก (Accommodation) — ข้อความก่อน "เข้า" จะถูกบันทึกเป็นที่พัก เช่น พระราม3 เข้า สวนหลวง
🚫
หากพิมพ์ "ออง" แต่ฐานข้อมูลมีแค่ "อองจี" — ระบบจะไม่บันทึกเพื่อป้องกันความผิดพลาด (ต้องพิมพ์ให้ตรงเป๊ะ)

📸 ลงเวลาด้วยรูปภาพ

ส่งรูปภาพพนักงาน (ใบบัตรรหัส/QR Code) ลงใน LINE กลุ่ม ระบบ AI Gemini Vision จะ:

  1. แกะรหัสพนักงาน 5 หลักจากรูป
  2. จับคู่กับฐานข้อมูลพนักงาน
  3. บันทึกเวลาโดยอัตโนมัติ
💡
รูปภาพต้องชัดเจน มีรหัส 5 หลักที่มองเห็นได้ ระบบรองรับทั้ง JPEG และ PNG

🤖 เครื่องมือ AI สำหรับผู้ใช้ทั่วไป

ระบบมี AI Tools 4 ประเภท สำหรับช่วยเหลือการทำงานประจำวัน:

1. 📝 สรุปข้อความ

วางข้อความยาวๆ แล้ว AI จะสรุปให้กระชับ เหมาะสำหรับรายงานการประชุมหรือเอกสาร

2. 🌐 แปลภาษา

แปลข้อความระหว่างไทย/อังกฤษ/พม่า เหมาะสำหรับสื่อสารกับแรงงานต่างชาติ

3. ✍️ ร่างเอกสาร

AI ช่วยร่างเอกสารตามคำสั่ง เช่น ร่างจดหมาย, ร่างรายงาน, เขียนข้อกำหนด

4. 💬 แชท AI

คุยกับ AI ถาม-ตอบ เรื่องทั่วไปหรือเกี่ยวกับงานก่อสร้าง

🤖 เครื่องมือ AI สำหรับ Admin

👑
เมนู "AI Admin" ต้องมีสิทธิ์ Admin เท่านั้น

เครื่องมือที่มี:

  • 📊 วิเคราะห์ KPI พนักงาน — ประเมินผลงาน OT, วันทำงาน
  • 📋 สร้างรายงานผู้บริหาร — สรุปภาพรวมโปรเจกต์อัตโนมัติ
  • 💡 วิเคราะห์ต้นทุน — ประเมินค่าใช้จ่ายจากข้อมูลชั่วโมงงาน
  • 📈 พยากรณ์แรงงาน — คาดการณ์ความต้องการพนักงานในอนาคต

👤 จัดการพนักงาน (CRUD)

Admin สามารถเพิ่ม/แก้ไข/ลบ ข้อมูลพนักงานผ่าน Dashboard ได้โดยตรง:

เพิ่มพนักงานใหม่

1

ไปที่เมนู "จัดการพนักงาน"

คลิกปุ่ม "+ เพิ่มพนักงาน" มุมบนขวา

2

กรอกข้อมูล

คำนำหน้า, ชื่อ, นามสกุล, แคมป์/ที่พัก, รหัส 5 หลัก, สถานะ

3

กด "บันทึกพนักงานใหม่"

ระบบจะตรวจสอบรหัสซ้ำก่อนบันทึก

เปลี่ยนสถานะพนักงาน

คลิกปุ่ม ปกติ / ลาออก ที่แถวของพนักงาน เพื่อสลับสถานะ

ลบพนักงาน

คลิกไอคอนถังขยะ 🗑️ แล้วยืนยันการลบ

📊 คลังข้อมูลและ Export

ค้นหาข้อมูล

พิมพ์ชื่อพนักงานหรือไซต์งานในช่องค้นหา ระบบจะกรองผลลัพธ์แบบ Real-time

Export CSV

คลิกปุ่ม "Export CSV" เพื่อดาวน์โหลดข้อมูลเป็นไฟล์ CSV สำหรับเปิดใน Excel

✅
ไฟล์ CSV รองรับภาษาไทย (UTF-8 with BOM) เปิดใน Excel ได้ถูกต้องทุกเครื่อง

⚙️ ตั้งค่าระบบ (Admin Only)

การตั้งค่าจะอัปเดตผ่าน Google Apps Script PropertiesService แบบ Real-time:

ค่า Configคำอธิบายค่าเริ่มต้น
SYSTEM_STATUSสถานะบอท ON/OFFON
IS_TESTINGโหมดทดสอบ TRUE/FALSEFALSE
BACKDATE_LIMITจำนวนวันย้อนหลังที่อนุญาต2
FUZZY_THRESHOLDค่าความแม่นยำจับคู่ชื่อ (0-1)1.0 (Exact Match)
💡
เปลี่ยน FUZZY_THRESHOLD เป็น 0.85 หากต้องการให้ระบบเดาชื่อใกล้เคียง (ไม่แนะนำสำหรับการใช้งานจริง)

📂 ไฟล์รายเดือน (MONTHLY_FILE_IDS)

ระบบอ้างอิงไฟล์ Google Sheets แยกตามเดือนในไฟล์ Config.gs:

var MONTHLY_FILE_IDS = [
  "xxx...",   // มกราคม (index 0)
  "xxx...",   // กุมภาพันธ์ (index 1)
  ...
  "xxx..."    // ธันวาคม (index 11)
];
🚫
หากเดือนใดไม่มี File ID ระบบจะแจ้ง "ไม่พบไฟล์ Google Sheets สำหรับเดือนนี้" — กรุณาเพิ่ม ID ใน Config.gs

❓ คำถามที่พบบ่อย

Q: ทำไมข้อมูลไม่บันทึกลงชีต?

A: ตรวจสอบว่า:

  • ไฟล์เดือนนั้นๆ มี File ID ใน MONTHLY_FILE_IDS
  • ไฟล์มีแท็บชื่อวันที่ภาษาไทย เช่น 16 มิถุนายน 2569
  • ไม่ได้ลงเวลาย้อนหลังเกินโควต้า

Q: ทำไมแสดง "Guest" แทนชื่อผู้ใช้?

A: เกิดเมื่อเปิด URL โดยตรงแทนการเปิดผ่าน Web App ของ Google Apps Script ระบบต้องเปิดผ่าน https://script.google.com/macros/s/xxx/exec

Q: โลโก้ไม่แสดง?

A: ตรวจสอบว่า URL รูปภาพ https://i.ibb.co/N3w3P9pS/52.png ยังใช้งานได้ หรือเปลี่ยนค่า LOGO_URL ในโค้ด

Q: LINE Bot ไม่ตอบ?

A: ตรวจสอบ:

  • SYSTEM_STATUS ต้องเป็น ON
  • LINE Webhook URL ถูกต้อง
  • Channel Access Token ไม่หมดอายุ

Q: ชื่อพนักงานหาไม่เจอ?

A: ระบบตั้งค่า Exact Match 100% — ต้องพิมพ์ชื่อตรงเป๊ะกับฐานข้อมูล ตรวจสอบสะกดให้ถูกต้อง

Q: เปลี่ยน Admin ได้อย่างไร?

A: แก้ไขอีเมล Admin ใน Config.gs → ADMIN_EMAILS array
