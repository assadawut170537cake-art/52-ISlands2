# Project Deduplication & Refactor

## Overview
Skill นี้ใช้สำหรับการตรวจสอบและทำความสะอาดทั้งโปรเจกต์ (Project-wide Code Refactoring) โดยมุ่งเน้นไปที่การค้นหาฟังก์ชันที่มีการทำงานซ้ำซ้อนหรือคล้ายคลึงกันที่ถูกเขียนแยกย้ายกันไปตามไฟล์ต่างๆ AI จะทำการรวบรวมฟังก์ชันเหล่านี้ ผสานกันเพื่อความสมบูรณ์ นำไปจัดเก็บไว้ที่ส่วนกลาง จากนั้นทำการเคลียร์โค้ดเดิม และช่วยอัปเดตไฟล์เหล่านั้นขึ้นระบบคลังข้อมูลด้วยระบบ Git และ Clasp

## Dependencies
Skill นี้ใช้เครื่องมือระดับ Agent เพื่อให้ทำงานแบบอัตโนมัติ 100%:
- `grep_search`: ค้นหารายชื่อฟังก์ชันและตัวแปร
- `view_file`: เจาะลึกวิเคราะห์เนื้อหาของฟังก์ชันเพื่อหาจุดซ้ำซ้อน
- `replace_file_content` / `multi_replace_file_content`: สำหรับลบ/แก้ไขโค้ด
- `run_command`: รันระบบ Version Control (`clasp push` และ `git`)

## Quick Start
วิธีการเรียกใช้งาน:
> "/project-deduplicate-refactor ตรวจสอบทั้งโปรเจกต์นี้ให้หน่อย"

## Workflow

### 1. Discovery (สำรวจระบบ)
- ใช้เครื่องมือ `grep_search` พิมพ์ค้นหาด้วย regex `^function ` หรือ `^const ` ควบคู่กันทั่วทั้งโฟลเดอร์ทำงาน (Workspace)
- ทำบัญชีรายชื่อฟังก์ชันและที่อยู่ของไฟล์ (Function Map) เพื่อเช็คจุดที่มีโอกาสซ้ำซ้อน

### 2. Analysis & Comparison (วิเคราะห์เชิงลึก)
- ตรวจจับฟังก์ชันที่: 
  - ชื่อเหมือนกันเป๊ะ (Duplicate Names)
  - ชื่อคล้ายคลึงกันมาก (เช่น `checkOT()` และ `checkOTAndProceed()`)
- อ่านเนื้อหาโค้ด ถ้าเป็นลอจิกเดียวกันเกิน 80% ให้จัดการมาร์กสถานะว่าต้อง "ผสาน" (Merge Candidate)

### 3. Refactoring & Merging (ผสานโค้ดและปรับโครงสร้าง)
- **สร้างโค้ดแกนหลัก (Master Function):** ผสานลอจิกที่มีประสิทธิภาพสูงที่สุดของทุกฝ่ายเข้าด้วยกัน
- **รักษารูปแบบ (No Minification):** ห้ามบีบอัดหรือย่อโค้ดเด็ดขาด ปรับโครงสร้างให้อ่านง่าย เพิ่มช่องไฟ (Indentation) และรักษาระบบคอมเมนต์เดิมไว้
- นำโค้ดที่ผสานเสร็จแล้วย้ายไปอยู่ในไฟล์ส่วนกลาง (เช่น `3_SharedFunctions.js`)

### 4. Cleanup (เคลียร์พื้นที่)
- กลับไปไล่ตามลบฟังก์ชันเก่าออกจากไฟล์ต้นฉบับต่างๆ เพื่อไม่ให้เปลืองพื้นที่
- หากพบไฟล์ไหนที่ฟังก์ชันโดนย้ายไปหมดแล้ว (ไฟล์ว่าง) ให้จัดการลบไฟล์นั้นออกจากระบบการจัดเก็บ หรือเสนอให้ผู้ใช้ลบทิ้ง

### 5. Version Control & Push (ส่งโค้ดขึ้นระบบ)
- รัน `run_command` เพื่อขออนุมัติใช้งานคำสั่ง:
  1. `clasp push` (สำหรับการอัปเดตโปรเจกต์ Google Apps Script)
  2. `git status` (เพื่อดูการเปลี่ยนแปลงไฟล์ลบ/เพิ่ม)
  3. `git add .` และ `git commit -m "Refactor: Merge duplicate functions & cleanup"`
  4. `git push`
- สรุปผลรายงานให้ผู้ใช้เห็นชัดเจนว่าไฟล์ใดถูกลบ ไฟล์ใดถูกเพิ่ม และมีฟังก์ชันกี่ตัวที่ถูกผสานไปแล้ว

## Common Mistakes
- **ลบผิดตัว:** ฟังก์ชันชื่อเหมือนกันแต่วัตถุประสงค์ต่างกันโดยสิ้นเชิง (เช่น `doPost` ในคนละโฟลเดอร์)
- **ลืมแก้ชื่อผู้เรียกใช้:** ถ้าย้ายฟังก์ชันแล้วมีการเปลี่ยนชื่อหรือรวมพารามิเตอร์ อย่าลืมอัปเดตโค้ดบรรทัดที่เรียกใช้ฟังก์ชันเหล่านั้นด้วย ไม่อย่างนั้นระบบพัง
- **รันคำสั่งลบไฟล์ก่อนที่ User จะยินยอม:** ต้องรอให้ผู้ใช้ตรวจงานก่อน หากแน่ใจแล้วถึงค่อยทำการ Push และเคลียร์ของเก่า
