# Apps Script Refactor & Merge

## Overview
Skill นี้ใช้สำหรับการผสานโค้ดข้ามไฟล์ในโปรเจกต์ Google Apps Script (GAS) โดยจะเปรียบเทียบฟังก์ชันระหว่างไฟล์เก่าและไฟล์เป้าหมาย หากพบฟังก์ชันที่ขาดหายไปจะดึงมาเติมให้สมบูรณ์ พร้อมทั้งสแกนหาข้อผิดพลาดทางโครงสร้าง เช่น การใช้ `async/await` อย่างไม่ถูกต้อง ซึ่งอาจทำให้เกิด Error 500 หรือ Timeout ในฝั่ง Webhook ได้ 

## Dependencies
Skill นี้เป็นรูปแบบ Instruction-Only ซึ่งจะใช้เครื่องมือพื้นฐานของ Agent เท่านั้น:
- `view_file` สำหรับอ่านโค้ดทั้งสองไฟล์
- `grep_search` สำหรับค้นหาตัวแปรและฟังก์ชันอย่างรวดเร็ว
- `replace_file_content` สำหรับแก้ไขและผสานโค้ดลงไฟล์เป้าหมาย

## Quick Start
คุณสามารถเรียกใช้ Skill นี้ได้โดยพิมพ์คำสั่งลักษณะนี้:
> "@[ไฟล์เป้าหมาย.js] /apps-script-refactor-merge @[ไฟล์ต้นฉบับ.js]"
> หรือ 
> "ใช้ทักษะ apps-script-refactor-merge ผสานโค้ดจาก A ไป B"

## Workflow

### 1. Analyze (วิเคราะห์)
- ใช้เครื่องมืออ่านและทำความเข้าใจโครงสร้างของไฟล์เป้าหมาย (ไฟล์ใหม่) และไฟล์แหล่งที่มา (ไฟล์เก่า)
- เช็คจุดเริ่มต้นของสคริปต์ (เช่น ฟังก์ชัน `doPost`, `doGet`) ว่ามีลำดับการประมวลผลอย่างไร

### 2. Compare (เปรียบเทียบ)
- สแกนหาฟังก์ชันที่เหมือนกัน แต่มีการเขียนลอจิกต่างกัน
- ค้นหาฟังก์ชันย่อยที่ถูกเรียกใช้งาน (Reference) ในไฟล์ใหม่ แต่ตัวฟังก์ชันจริงๆ ยังคงติดอยู่ในไฟล์เก่า (ยังไม่ได้ย้ายมา)
- ระบุลอจิกที่ดีกว่าและตัดสินใจว่าจะดึงฟังก์ชันไหนเป็นหลัก

### 3. Refactor & Fix (ปรับแก้ลอจิกและโครงสร้าง)
- **จำกัด Async/Await:** ตรวจสอบการใช้งาน `async/await` หากมีการดึงข้อมูลผ่าน API ด้วย `UrlFetchApp` ใน GAS ลอจิกนั้นจะเป็น Synchronous อยู่แล้ว การใส่ `async` ที่ตัวฟังก์ชันโดยไม่จำเป็นมักจะทำให้ `doPost` คืนค่าเป็น Promise และระบบพัง ให้จัดการถอดออกให้หมด
- **Guard Clauses:** ตรวจสอบเงื่อนไขด่านหน้า (เช่น การเช็ค Whitelist, เช็ค Admin) เพื่อดูว่ามีคำสั่งใดถูกบล็อกหรือเผลอ Return ทิ้งก่อนถึงเวลาอันควรหรือไม่

### 4. Merge (ดำเนินการผสาน)
- ใช้เครื่องมือเพื่อปรับแต่งไฟล์เป้าหมายให้สมบูรณ์
- **ข้อควรระวัง:** หากลอจิกที่ต้องแก้มีขนาดใหญ่ ให้ใช้วิธีทยอยผสาน (Chunk by Chunk) แทนที่จะทำทั้งหมดในรวดเดียว เพื่อความปลอดภัย

### 5. Verify (สรุปและตรวจสอบ)
- เมื่อผสานเสร็จสิ้น ให้ทบทวนการอ้างอิงชื่อฟังก์ชันทั้งหมดว่าตรงกันและไม่มีใครพิมพ์ชื่อฟังก์ชันผิด
- สรุปผลลัพธ์เป็น Walkthrough ให้ผู้ใช้งานรับทราบว่ามีฟังก์ชันไหนถูกลบ ถูกเพิ่ม หรือถูกซ่อมแซมบ้าง

## Common Mistakes
- **ลืมเช็คขอบเขต (Scope):** ย้ายโค้ดมาแต่วางผิด Scope ทำให้ตัวแปร Local กลายเป็น Undefined
- **ลบ/แก้ Guard Clause ผิดที่:** ทำให้สคริปต์พังหรือสูญเสียระบบความปลอดภัย
- **ไม่ได้ทดสอบ:** ผสานเสร็จแล้วไม่ได้ให้ผู้ใช้งานลอง Push เพื่อเช็ค Syntax error
